# Dig制度 P4（バックエンド結合）技術設計

要件定義 v1.1/v1.2 の P4。DB永続化・API・監査ログ・CI・デプロイ。dgloss `TECH_STACK`/`CONVENTIONS` 準拠。

## 1. データベース（Supabase / Prisma）
- 本番: **Supabase(PostgreSQL・ap-northeast-1)**、ローカル: Postgres。接続は `DATABASE_URL`（サーバ専用）。
- `prisma/schema.prisma`: Setting / SalaryRange / Member / MonthlyEvaluation / BonusDigItem / BonusDigRecord / Loan / Transaction / **AuditLog**。金額 Decimal・enum 日本語。
- マイグレーション: `pnpm db:migrate`（初回 `init` 適用済）。生成: `pnpm db:generate`。
- シード: `pnpm db:seed`（`prisma/seed.ts`・dig_v5.xlsx のメンバー＋設定＋ボーナス項目＋入社時初回借入。月次評価は `@dig/core` で計算投入）。

## 2. API（Next.js Route Handlers・zod・監査ログ）
`apps/web/app/api/*`（すべて `runtime=nodejs` / `dynamic=force-dynamic`）。

| メソッド・パス | 用途 |
|---|---|
| GET `/api/health` | DB接続確認 |
| GET `/api/members` | メンバー一覧 |
| GET `/api/evaluations?ym=` | 月次評価 |
| GET `/api/loans` | 借入一覧 |
| POST `/api/loans/[id]/decision` | 借入 承認/却下（`{approve, actor}`） |
| GET `/api/settings?ym=` | 設定 |
| PATCH `/api/settings/rate` | 会社金利の変更（`{yearMonth, annualRatePct, actor}`） |
| GET/POST `/api/bonus?ym=` | ボーナスDig 項目＋記録 / 記録追加 |
| GET/POST `/api/transactions?ym=` | 送金一覧 / 送金追加 |

- 入力は **zod** 検証（`@dig/contracts` のスキーマ再利用）。HTTPコード: 必須欠落=400 / 未存在=404 / 競合=409。エラーにスタックトレースを含めない。
- 変更系（承認・金利・送金・ボーナス）は **AuditLog** に記録（actor/action/entity）。
- `serialize()` で Prisma Decimal→number・Date→ISO へ整形（Decimal.toJSON の文字列化を回避）。
- rawSQL 不使用（Prisma 標準クエリのみ）。

## 3. UI 結合
- **金融管理（FinanceConsole）を実APIに接続**: `/api/loans`・`/api/members`・`/api/settings` を読み、承認/却下は `POST /api/loans/[id]/decision`、金利変更は `PATCH /api/settings/rate` で永続化。DB未接続時はモックへ自動フォールバック（`● DB接続` / `○ モック表示` バッジ）。
- 他タブ（予実モニター/メンバー評価/Digloss Bank/ボーナスDig/取引ログ）は現状モック表示。API endpoint は整備済のため、同パターンで順次結合可能。

## 4. CI / リリース（dgloss RELEASE.md 準拠）
- `.github/workflows/ci.yml`: install → prisma generate → contracts/core build → `pnpm -r typecheck` → core test → web build。
- `.github/workflows/release.yml`: main push で **semantic-release**（Conventional Commits から SemVer・CHANGELOG・tag・GitHub Release を自動生成）。`.releaserc.json`。**版数・CHANGELOG は手書きしない**。

## 5. デプロイ（Vercel dg-bo）
- `vercel.json`: monorepo ビルド（contracts→core→prisma generate→web build）。
- 必要な環境変数（Vercel/Supabase 側で設定）: `DATABASE_URL`（＋認証用 Google/Supabase Auth）。
- APX(AWS) とは別プラットフォーム。本番波及なし。
