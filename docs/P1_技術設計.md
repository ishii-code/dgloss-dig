# Dig制度 P1（コア）技術設計

- **対象**: 要件定義 v1.1 の P1（設定/メンバーマスタ・月次評価・成果Dig・予算Dig・達成率/ランク・ダッシュボード）
- **標準準拠**: dgloss `TECH_STACK.md` / `CONVENTIONS.md` / `BRAND.md` / `ARCHITECTURE.md`

## 1. アーキテクチャ（contract-first / monorepo+packages）
小規模・型跨ぎ変更が多いため、`ARCHITECTURE.md §5` に従い **pnpm monorepo + packages** を採用（polyrepo の細分割を代替）。結合点は `@dig/contracts` に一本化。

```
dgloss-dig/
├─ packages/
│  ├─ contracts/   @dig/contracts … 型・zodスキーマ・enum（source of truth・依存なし）
│  └─ core/        @dig/core      … 計算エンジン（純関数）+ Vitest（依存: contracts）
├─ apps/
│  └─ web/         @dig/web       … Next.js(App Router) ダッシュボード（依存: contracts, core）
├─ prisma/         schema.prisma  … Supabase(PostgreSQL) データモデル
└─ docs/
```

依存グラフ: `contracts → core → web`。DBは境界内に閉じる（web/api のみ Prisma 経由）。

## 2. 技術スタック（TECH_STACK.md プロファイルA）
TypeScript(strict) / Node 22 / pnpm / Next.js App Router / Tailwind + shadcn/ui 方針 / Supabase(PostgreSQL)+Prisma / zod / Vitest / Vercel(dg-bo)。LLMは将来利用時 Anthropic Claude 統一。

## 3. 計算エンジン（@dig/core）
要件 §7 の数式を純関数で実装。主な関数:
`residencyDays` / `prorationCoefficient` / `seatCost` / `totalCost` / `mround`(Excel MROUND互換) / `monthlyBudgetDig` / `cumulativeBudgetDig`(四半期×3・半期×6) / `actualDig` / `achievementRate` / `evaluationRank`(S/A/B/C/D) / `evaluateMonthly`(集約) / `loanSchedule`(元利均等) / `computeQuarterBalance` / `incentiveAmount`(×20%) / `promotionStep`。

## 4. データモデル（Prisma / Supabase）
`prisma/schema.prisma`: Setting / SalaryRange / Member(personId=手入力PersonID) / MonthlyEvaluation(確定後イミュータブル) / BonusDigItem / BonusDigRecord / Loan / Transaction。金額は `Decimal`。enum は日本語ラベル（EmploymentType/EvaluationCycle/Position/JobType/Rank/MemberStatus）。

## 5. UI（@dig/web・BRAND.md 準拠）
dgloss 経営AI OS ダッシュボードにトーンを合わせる。
- カラー: Primary `#2563EB` / Accent `#7C3AED` / 白基調（PECO黄橙は不使用）。数値は `tabular-nums`。
- 画面: ヘッダ（ロゴ左上）→ タブナビ（アクティブ青下線）→ フィルタ（対象Q・単月/累計）→ 全社KPI大カード（実績Dig/インセン・進捗バー）→ 事業部別カード → メンバー評価テーブル（ランクバッジ）。
- 状態表示（ローディング/エラー/空）は P1 で最小、P2 で拡充。成果Digは手入力運用（v1.1 Q3）。
